Deep Neural
Networks
Recognizing digits is easy to do for a
human, but it is difficult to explain 
to a computer what are the steps to
do it.
The normal algorithmic approach
doesn't work, you need to do it in a
different way.


There are a lot of Pattern recognition techniques to do this,
but currently Neural Networks achieve the best accuracy in these
tasks, for example in the MNIST challenge of recognizing
handwritten digits, the top results are based on them.
( http://yann.lecun.com/exdb/mnist/ )
The main idea
We want to emulate the structure of the human brain
(since it's actually really good at tasks like this) and use it
to solve this problem.

Also we want to develop it such that it can automatically learn
from examples (just like humans learn to read).
The set of examples combined with the correct interpretation
is called training set.

Since we are trying to emulate the human brain, it stands to 
common sense that we first need to define an equivalent
for the neuron.
Perceptrons are very simple:
  - a set of binary inputs x
  - a single binary output
  - a set of weights w 
    (one for each input)
  - a threshold value
such that




The basic idea is that a perceptron makes decisions weighing up
different evidence. You can regulate the decision process by
tuning the values of w and threshold.
It stands to reason that you can combine several perceptrons
to make more complex decisions
Perceptrons
We will discuss only feedforward neural networks, in which we have
different layers of neurons and the output of the neurons in a layer
is the only input of the neurons of the next layer, until the final one,
which is the output.
There are recurrent neural networks, in which loops are possible,
and so they are more similar to human brains, but to date they still
have no efficient learning algorithms.

Learning
We know that Neural Networks are at least as powerful as circuits,
but is that all? No.
Neural networks are radically different from logic circuits, because
instead of having to design them, you can make them learn to solve 
problems by using algorithms that tune the weights and the biases
in response to external stimuli (such as a training set).
We can see it as optimizing some multidimensional function in 
which the inputs are the weights and the biases.
But to use already estabilished optimization techinques we need 
an important property which Perceptrons don't satisfy:




It's easy to see that a perceptron can switch from 0 to 1 with a small
change of a weight and cause a snowball effect in the net.
"Small changes in weights (or biases) cause
  small changes in the output"
Sigmoid neurons
We introduce a new type of neurons which satisfy our continuity
property: sigmoid neurons.
The first difference is that now the inputs x1, x2, ... are not just
0 and 1 but any value between them (xi€[0,1]).
The second difference is that the output of the neuron is now:



also which is the output of the σ function:



(So the output is σ(w⋅x+b)).
If z=w⋅x+b is a large positive number then σ(z)≈1.
If z=w⋅x+b is a large negative number then σ(z)≈0.
Sigmoid function
You can easily see that is a smoothed version of the step function 
which represents the output of a perceptron.
With this function, a small change in the weights and the biases
causes a small (linear in the change of w and b) change in the output.
So we can use optimization techniques on a net of sigmoid neurons.
Since the ouput is now a number between 0.0 and 1.0 we decide to
round it to the nearest integer as a convention.
Training the neural network
Since we do training by minimizing a function, we define the
cost function as follows:


  • "a" is short for a(w,b,x) and it's the output of the network.
  • "y" is the output we want from that input x
  • "y" and "a" are arrays of 0s and 1s and we want to minimize

So we want to find weights and biases and make C(w,b)≈0.
Obviously we can define different cost functions, this is not
the only one.
We are going to look for the values of the vectors w and b that
minimize the function. We are going to use gradient descent.
(which is an approximate techique, because it is too complicate

Gradient descent
The idea behind gradient descent is simple: start from a random
point and follow the function in the direction which decreases
the value of the function to minimize the most.

The algorithm consists of repeating the step v→v′=v−η∇C until 
convergence to a global minimum (we hope!).
Of course we will have to choose the value of the learning rate:
too big and we risk to get a ΔC>0, too small and it will take a lot of
iterations to reach convergence.

Applied to our own case the steps will be:
  • w(k)→w′(k)=w(k)−η*(∂C/∂w(k))
  • b(l)→b′(l)=b(l)−η*(∂C/∂b(l))
But how do we compute ∇C?
∇C is the average of the cost of each training input.
So we need to compute ∇C(x) for every training input x and then
average them to obtain ∇C. But since we need a lot of samples to
train a network correctly, it's going to take a long time.
The solution is using stochastic gradient descent which basically
consists in computing ∇C using only a randomly chosen subset
of the training set different for each epoch of training.
