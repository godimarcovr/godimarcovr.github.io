Deep Neural
Networks
Why Neural Networks?

  • Some tasks are easy for humans, but difficult for computers
    (e.g. recognizing digits)

  • The problem lies in the use of the "classical" algorithmic approach

  • In Pattern Recognition neural networks currently achieve the
    best results (e.g.     in the MNIST 
    challenge the best scores
    are NNs)

  • Can we use Neural Networks
    to solve Artificial Intelligence
    tasks that are normally
    difficult to solve?
The main idea
  • To simulate the structure of the human brain
  • To make it learn from examples (training set)
Since humans are so good at learning to do tasks like this we want


We want to build a brain, so we first need to define the neuron
Perceptrons
  • A set of binary inputs x
  • A set of weights w (|w|=|x|)
  • A threshold value
  • A single binary output a




Evidence is weighted up and used to make a decision.

By changing weights and threshold, you can tune the decision rules.

By using multiple perceptrons (serial and/or parallel) you can make
more complex decisions.
We are going to focus on feedforward neural networks.

Recurrent neural networks also exist, but while interesting they
are not practical yet, but they are being studied.
Computational power of Neural Networks
Simulation of a NAND port with a neuron:







If a neuron is a port, a neural network is a circuit.
We get the intuition that they are at least as powerful as a circuit
Learning
  • We don't need to manually design NNs like we do with circuits
  • We can make them to solve our problems by finding good values
    for the weights and biases
  • We can do this in an automatic way by providing a training set to
    an optimization algorithm
A property that we need to use already estabilished techniques is

"Small changes in weights (or biases) cause 
                                                        small changes in the output"

Perceptrons don't satisfy this.
Sigmoid Neuron
New type of neuron:
  • Input and output values are now real numbers
    between 0.0 and 1.0
  • Output value is now the value of σ(w⋅x+b)






  • Now, a small change in the parameters causes a small change
    in the output
Smoothed version of a step function
Training
Minimize a cost function:

Too complicated to solve analitically.

Use gradient descent:





     Repeat the last two steps until convergence.
  • Start from a random initialization
  • Calculate gradient
  • Make a step (η called learning rate) towards the direction of
    maximum decrease (opposite of gradient)
The cost function is an average of the cost of every training input

But we have thousands of samples,
it's too costly.

So we are going to use stochastic
gradient descent.
Basically estimate gradient using only a
randomly chosen subset
of the training set.


Training set size M  and subset size is m (m<<M)
Example MNIST
M=50000
m=10
It's a 5000x speedup at a cost of a little precision in the estimate
(complexity is M*C(x) or m*C(x))
Backpropagation
  • Fast algorithm to compute gradient
  • Fundamental in deep neural networks
    (made deep networks possible)

  • Cost of a single sample is a function of the network outputs

What is the complexity?





The complexity is O((W+B)*C)

a complexity of O(2*C)

Example
In the relatively simple problem of MNIST, the author uses a
neural network of [784,30,10] which has 23820 weights.
In this little example we have a speedup bigger than 20000x!
  • C cost of computing the network output (polynomial on W and B)

More improvements
  • Cross Entropy: a different cost function that makes the network
                                       learn faster when the error is big

  • Softmax: turn outputs into a probability distribution and use a
                         log-likelihood cost function

  • L2 regularization: add a term to the cost function to keep
                                             weights at low values

  • Better random initialization of the parameters

  • Choosing hyper-parameters automatically using validation data

  • Increasing the training data
Applications in Artificial Intelligence
Feedforward Neural Networks, used in: Computer Vision, Pattern
Recognition, Natural Language Processing, Speech Recognition
and so on.


A rule of thumb: they perform well on tasks for which we don't have
a solid algorithmic approach.


Problems studied in class are probably better with a "classical"
approach, rather than NNs.



They are useful in situations where "human intuition" is good at.
Artificial Intelligence in games
  • 1997 IBM's DeepBlue defeats chess world champion
    Garry Kasparov

  • 2015 Google's AlphaGo defeats Lee Sedol, one of the best Go players
While DeepBlue was a remarkable result, it did not cause some 
kind of "revolution" in AI.

What about AlphaGo?
DeepBlue
  • Express chess knowledge with a function that gives a value for
    every board state
  • Explore the choice tree and choose the branch that gives
    best results based on the function
The algorithm structure was not much different from earlier
chess AIs.

It used cleverer heuristics and had 
much better hardware, but it
was hardly something new.

The "knowledge" used in the 
evaluation function was coded in.
AlphaGo
Go is more difficult than chess:




Human intuition was still better than AIs at evaluation.
  • A lot more possible moves
  • Difficult to evaluate positions
AlphaGo started with a neural network trained with training set of
150000 games played by professionals.

The network was able to predict which move a human player would
make.
It could play, but it was still at amateur level.

It was then improved by making it play against itself.
(Reinforcement Learning)
This network is called the policy network.

Tested against Pachi (a search based Go AI), it won 85% of 
the games played and it was much faster!


But it wasn't enough.
By using as a training set all of the positions reached through
policy network self-play, a value network was trained.

It was able to give the likelihood of a win given the current state.

It's like an evaluation function, but trained instead of learned.

This function was used to guide a Monte Carlo Tree Search.


The innovations introduced by AlphaGo are:





This is important, because the neural network was somehow able
to "bottle" the intuition needed to play this game!
  • The Monte Carlo Tree Search (a new kind of heuristic tree search)
  • An evaluation function trained and not designed by hand
Deep Q

Recurrent Neural Networks
  • Activations of neurons influenced by activation at earlier times

  • Input to the network not only the current input

  • Useful for modeling change over time

  • Short term memory

  • Applications in Natural Language Processing

  • Speech recognition (records in phoneme recognition)
